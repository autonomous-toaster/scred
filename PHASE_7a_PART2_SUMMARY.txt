TECHNICAL DETAILS
═══════════════════════════════════════════════════════════════════════════════

Configuration Loading Precedence (6 levels):
  1. /etc/scred/config.yaml (system-wide)
  2. ~/.scred/config.yaml (user home)
  3. scred-{ENV}.yaml (environment-specific, e.g., scred-production.yaml)
  4. ./scred.yaml (current directory)
  5. $SCRED_CONFIG_FILE (environment variable override)
  6. Built-in defaults (if no file found)

Per-Path Redaction Rules (scred-proxy exclusive feature):
  
  Example:
  rules:
    - path: "/health"
      redact: false
      reason: "Health check endpoint"
    
    - path: "/api/internal/*"
      redact: false
      reason: "Internal API, no redaction needed"
    
    - path: "/logs/*"
      redact: true
      patterns:
        detect: [CRITICAL]
        redact: [CRITICAL]

Path Matching Logic:
  - Exact match: "/health" matches "/health" only
  - Prefix wildcard: "/api/internal/*" matches "/api/internal/users", etc.
  - Dual wildcard: "*/logs/*" matches "/app/logs/error.log", etc.
  - All match: "*" matches any path

USAGE EXAMPLES
═══════════════════════════════════════════════════════════════════════════════

1. Load from config file:
   $ scred-proxy
   # Automatically loads from ./scred.yaml or ~/.scred/config.yaml

2. Use environment-specific config:
   $ SCRED_ENV=production scred-proxy
   # Loads config-production.yaml

3. Override with env vars:
   $ SCRED_PROXY_LISTEN_PORT=8888 scred-proxy
   # Uses env var if no config file

4. Load from specific file:
   $ SCRED_CONFIG_FILE=/etc/scred/production.yaml scred-proxy
   # Uses specified config file

5. With per-path rules:
   Config file has:
   rules:
     - path: "/health"
       redact: false
     - path: "/api/internal/*"
       redact: false
   # scred-proxy automatically applies rules
