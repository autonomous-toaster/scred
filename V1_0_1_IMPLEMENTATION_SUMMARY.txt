CHANGES DETAIL
════════════════════════════════════════════════════════════════════════════

FIX #1: Proxy Per-Path Rules Now Actually Work

BEFORE:
  • Rules were parsed from config
  • Rules were stored in ProxyConfig
  • Rules were NEVER checked during request processing
  • All requests were redacted according to global mode
  • Per-path rule feature was dead code!

AFTER:
  • Extract request path from HTTP request line
  • Check per-path rules BEFORE redaction
  • Skip redaction if rule says don't redact path
  • Apply redaction if rule says redact path
  • Use default mode if no matching rule
  • Log which rule was applied

EXAMPLE:
  Config: rules:
            - path: "/health*"
              redact: false
  
  Request: GET /health
  Result: ✅ Redaction SKIPPED (rule matched)

───────────────────────────────────────────────────────────────────────────

FIX #2: Regex Selector Now Actually Does Regex

BEFORE:
  • --redact "regex:^sk-" was completely broken
  • Used string contains() instead of regex
  • Pattern name had to contain "^sk-" literally
  • Result: NO patterns matched, redaction didn't work

AFTER:
  • Actual regex::Regex compilation
  • Proper pattern matching with anchors
  • Supports groups, lookahead, etc.
  • Graceful error handling for invalid regex
  • Pattern warnings logged (doesn't crash)

EXAMPLE:
  Command: scred --redact "regex:^sk-"
  Before: No patterns matched (broken)
  After: ✅ All patterns starting with sk- matched

───────────────────────────────────────────────────────────────────────────

FIX #3: Invalid Selectors Now Fail Immediately

BEFORE:
  $ scred --redact CRIITICAL
  [WARN] Invalid SCRED_REDACT_PATTERNS 'CRIITICAL': Unknown tier: criitical
  → Falls back to CRITICAL,API_KEYS,PATTERNS (silent!)
  → User thinks CRIITICAL is applied, but isn't!

AFTER:
  $ scred --redact CRIITICAL
  ERROR: Invalid SCRED_REDACT_PATTERNS value: 'CRIITICAL'
  Reason: Unknown tier: criitical

  Valid tier names: CRITICAL, API_KEYS, INFRASTRUCTURE, SERVICES, PATTERNS
  Valid patterns: aws-*, github-*, sk-*, etc.
  Valid regex: regex:^(aws|github)
  Examples:
    scred --redact CRITICAL
    scred --redact CRITICAL,API_KEYS
    scred --redact 'regex:^sk-'
  
  [Exit code: 1]
  → User IMMEDIATELY knows configuration is wrong!
