PHASE 7a: CONFIGURATION MANAGEMENT (MANDATORY)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

GOAL: Support YAML/TOML configuration files for all three binaries

REQUIREMENTS:
────────────

1. Configuration File Format
   • File locations:
     - /etc/scred/config.yaml (system-wide)
     - ~/.scred/config.yaml (user)
     - ./scred.yaml (current directory)
     - Precedence: CLI args > env var > ./.scred.yaml > ~/.scred/config.yaml > /etc/scred/config.yaml
   
   • Format: YAML primary, TOML alternative
   
   • Schema validation on load
   • Clear error messages for invalid configs
   • Type checking and defaults

2. Configuration Sections

   For scred-cli:
   ```yaml
   scred-cli:
     mode: auto              # auto | env | text
     streaming: true
     patterns:
       detect: [CRITICAL, API_KEYS, INFRASTRUCTURE, JWT, CUSTOM]
       redact: [CRITICAL, API_KEYS]
   ```

   For scred-proxy:
   ```yaml
   scred-proxy:
     listen:
       port: 9999
       address: "0.0.0.0"
     upstream:
       url: "https://backend.example.com"
     redaction:
       mode: selective        # selective | strict | passive
       patterns:
         detect: [CRITICAL, API_KEYS, INFRASTRUCTURE]
         redact: [CRITICAL, API_KEYS]
       rules:
         - path: "/api/internal/*"
           redact: false
           reason: "Internal API, no redaction needed"
         - path: "/logs/*"
           redact: true
           patterns: [CRITICAL]
   ```

   For scred-mitm:
   ```yaml
   scred-mitm:
     listen:
       port: 8080
       address: "0.0.0.0"
     upstream_proxy:
       enabled: false
       url: "http://corporate-proxy:8080"
     redaction:
       patterns:
         detect: [CRITICAL, API_KEYS, INFRASTRUCTURE]
         redact: [CRITICAL, API_KEYS]
     ca_cert:
       generate: true
       path: "/tmp/scred-ca.pem"
   ```

3. Environment Variable Overrides
   • ${VAR_NAME} substitution in config files
   • Override config values: SCRED_CONFIG_OPTION=value
   • Example: SCRED_PROXY_LISTEN_PORT=8888 overrides listen.port

4. Per-Path Redaction Rules (scred-proxy specific)
   • Path-based routing: /api/v1/*, /internal/*, /logs/*, etc.
   • Per-path pattern selection
   • Wildcard and regex support
   • Rule precedence and conflicts
   • Logging of which rule matched

5. Configuration Validation
   • Schema validation on startup
   • `--validate-config` flag for checking
   • `--dry-run` mode to test without applying
   • Exit with clear errors if invalid

6. Hot-Reload (scred-proxy/scred-mitm)
   • Watch config file for changes
   • Graceful reload without dropping connections
   • Signal support (SIGHUP = reload)
   • Rollback on validation error
   • Logging of reload events

7. Environment-Specific Configs
   • Load different configs per SCRED_ENV
   • SCRED_ENV=production → config-prod.yaml
   • SCRED_ENV=staging → config-staging.yaml
   • SCRED_ENV=dev → config-dev.yaml
