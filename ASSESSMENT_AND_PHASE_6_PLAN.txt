┌─────────────────────────────────────────────────────────────────────────────┐
│ SOLUTION: ConfigurableEngine (Phase 6)                                      │
└─────────────────────────────────────────────────────────────────────────────┘

Create a unified wrapper in scred-http that provides:

  pub struct ConfigurableEngine {
      engine: RedactionEngine,              // Core detection/redaction
      detect_selector: PatternSelector,     // Filter detection
      redact_selector: PatternSelector,     // Filter redaction
  }

  impl ConfigurableEngine {
      pub fn detect_only(&self, text: &str) -> Vec<DetectionWarning> { ... }
      pub fn redact_only(&self, text: &str) -> String { ... }
      pub fn detect_and_redact(&self, text: &str) -> FilteredResult { ... }
  }

Then all three binaries use ConfigurableEngine:
  • scred-cli: Replace ZigAnalyzer with ConfigurableEngine
  • scred-proxy: Replace raw engine.redact() with ConfigurableEngine
  • scred-mitm: Use ConfigurableEngine for consistency

RESULT: One code path for all binaries, consistent behavior everywhere
