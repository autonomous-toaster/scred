ARCHITECTURE DECISIONS

1. PatternSelector in scred-redactor (not scred-http)
   Reason: Avoids circular dependency
   - scred-http imports scred-redactor
   - Both can now import PatternSelector
   - Single source of truth

2. Optional selector approach
   Reason: Backward compatibility + type safety
   - selector: Option<PatternSelector>
   - None = use all patterns (existing behavior)
   - Zero runtime overhead for default case
   - Type system ensures selector always used when provided

3. TDD methodology
   Reason: Clear success criteria, testability
   - Write tests before implementation
   - Verify tests fail without implementation
   - Implement until all tests pass
   - Document future phases with #[ignore] tests
