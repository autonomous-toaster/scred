================================================================================
                              KEY INSIGHTS
================================================================================

1. DNS BACKOFF WAS EXCESSIVE
   - Original: 100ms, 200ms, 400ms delays
   - Problem: Even failed localhost connections delayed 100ms+ between retries
   - Solution: 1ms, 2ms, 4ms still exponential but much faster
   - Impact: 58.8% throughput improvement

2. LOGGING IS NOT THE BOTTLENECK
   - Hypothesis: Reduce logging with RUST_LOG=off
   - Result: Actually -23% slower (MADE IT WORSE)
   - Insight: Logging overhead is minimal; system needs diagnostic output

3. I/O IS NOT THE BOTTLENECK
   - Larger buffers (-26%): Contraindicate normal I/O tuning
   - TCP_NODELAY (-26%): Disabling Nagle made it worse
   - Insight: Bottleneck is computational (pattern matching), not I/O

4. CONCURRENCY REVEALS CONTENTION (53% regression)
   - 20 parallel requests: 0.008 MB/s vs 0.027 sequential
   - Root cause: No connection pooling, resource contention
   - Next optimization target: async improvements or pattern caching
