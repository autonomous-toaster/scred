CRITICAL BLOCKERS (v1.0.1 - 5 hours)
─────────────────────────────────────

Issue #1: PROXY PER-PATH RULES NOT ENFORCED (2h)
  Problem: Rules defined in config but never checked during redaction
  Impact: Configuration provides false sense of security
  Fix: Add path matching before redaction call
  Status: ✅ VERIFIED

Issue #2: REGEX SELECTOR COMPLETELY BROKEN (2h)
  Problem: Uses string contains() instead of regex
  Example: --redact "regex:^sk-" looks for literal "^sk-" in pattern names
  Fix: Implement actual regex matching
  Status: ✅ VERIFIED

Issue #3: SILENT FALLBACK ON INVALID SELECTOR (1h)
  Problem: User typo silently changes behavior (no error, just fallback)
  Example: --redact CRIITICAL (typo) falls back to default (includes API_KEYS)
  Fix: Exit with error instead of warn + fallback
  Status: ✅ VERIFIED

HIGH PRIORITY FIXES (v1.1 - 8 hours)
────────────────────────────────────

Issue #4: Inconsistent Selector Precedence (2h)
  Problem: CLI, MITM, and Proxy use different precedence orders
  Impact: Same config behaves differently on different components
  Fix: Unified precedence everywhere

Issue #5: Multiline Secrets Not Detected (2.5h)
  Problem: JWT spanning 3 lines → last 2 lines NOT redacted
  Impact: Partial secret exposure
  Fix: LineAccumulator for line continuations

Issue #6: Pattern Name Validation Missing (1h)
  Problem: Unknown pattern names silently default to generic tier
  Fix: Return error on unknown patterns

Issue #7: Error Handling Inconsistency (1.5h)
  Problem: Error messages differ across CLI/MITM/Proxy
  Fix: Centralized error handling module
