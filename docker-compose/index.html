🐳 SCRED Fake Upstream Service

This service hosts sensitive data files for testing SCRED's response-only redaction capabilities.

⚠️ Warning: This service intentionally exposes sensitive data patterns for testing purposes only. Do not use in production with real secrets.

Available Endpoints

GET /secrets.json - Returns JSON file with various sensitive data patterns
GET /index.html - This page

Sensitive Data Patterns Included

  • ✓ AWS IAM keys (AKIA... format)
  • ✓ API tokens (sk-... format)
  • ✓ GitHub tokens (ghp_, ghu_, glpat- formats)
  • ✓ Database credentials (PostgreSQL, MySQL, MongoDB)
  • ✓ JWT/OAuth tokens
  • ✓ Private keys (RSA, EC, OpenSSH)
  • ✓ GCP/Firebase service account keys
  • ✓ Stripe live keys (sk_live_ format)
  • ✓ Multiple secrets in single values
  • ✓ Base64-encoded secrets
  • ✓ Secrets in URL parameters
  • ✓ Secrets in JSON object values

Testing Scenarios

This fake upstream is designed to be used with SCRED's response-only redaction proxies:

Scenario 1: Direct Access (No Redaction)

$ curl http://localhost:8001/secrets.json
# Response: Raw secrets visible in JSON

Scenario 2: Through Reverse Proxy (Response Redacted)

$ curl http://localhost:9998/secrets.json
# Response: Same JSON structure, but all secrets replaced with 'x' characters

Scenario 3: Through MITM Proxy (Response Redacted)

$ curl --proxy http://scred-mitm-response-only:8889 \
  http://fake-upstream:8001/secrets.json
# Response: Same as reverse proxy - secrets redacted

Key Feature: Response-Only Redaction

Both the reverse proxy and MITM proxy are configured to:

  • NOT redact requests: Your requests pass through unmodified
  • REDACT responses: The upstream's response is filtered before returning to you

This is useful when:

  • Upstream APIs expose secrets in responses (e.g., API keys, tokens)
  • API documentation includes example credentials
  • Error messages contain sensitive information
  • You want to protect the client from seeing upstream's secrets

Character Preservation

SCRED maintains the same length when redacting secrets:

  • Input: AKIAIOSFODNN7EXAMPLE (20 chars)
  • Output: AKIAxxxxxxxxxxxxxx (20 chars)

Service Information

  • Port: 8001
  • Protocol: HTTP
  • Server: Python built-in HTTP server
  • Data file: secrets.json (read-only mounted volume)
