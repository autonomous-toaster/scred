PHASE 7a: Configuration Management
───────────────────────────────────

Current State:
  • Environment variables for basic config
  • CLI flags for pattern selection
  • Hardcoded defaults

New Features:
  1. YAML Configuration File
     File: /etc/scred/config.yaml
     
     Example:
     ```yaml
     scred-proxy:
       listen:
         port: 9999
         address: "0.0.0.0"
       upstream:
         url: "https://backend.example.com"
       redaction:
         mode: selective  # or: strict, passive
         patterns:
           detect: [CRITICAL, API_KEYS, INFRASTRUCTURE]
           redact: [CRITICAL, API_KEYS]
       rules:
         - path: "/api/internal/*"
           redact: false
           reason: "Internal API, no redaction needed"
         - path: "/logs/*"
           redact: true
           patterns: [CRITICAL]
           reason: "Logs only redact critical secrets"
     
     scred-cli:
       mode: auto  # or: env, text
       streaming: true
       patterns:
         detect: [CRITICAL, API_KEYS, INFRASTRUCTURE]
         redact: [CRITICAL, API_KEYS]
     ```

  2. TOML Alternative (for those who prefer it)
     File: /etc/scred/config.toml
     
  3. Configuration Validation
     • Schema validation on startup
     • Clear error messages for invalid config
     • Config check command: `scred-proxy --validate-config`
     
  4. Hot Reload
     • Watch config file for changes
     • Graceful reload without dropping connections
     • Rollback on validation error

  5. Environment-Specific Configs
     • Production config
     • Staging config
     • Development config
     • Load based on environment variable

Effort: 4-6 hours
